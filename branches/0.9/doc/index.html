Overview

Classes summary
agente El agente es el que se asocia a un albarán, factura o caja. Cada usuario puede estar asociado a un agente, y un agente puede estar asociado a varios usuarios.
albaran_cliente Albarán (boceto de factura o factura preliminar) de cliente
albaran_proveedor Albarán de proveedor (boceto de factura o factura preliminar)
almacen El almacén donde están físicamente los artículos.
articulo Representa el artículo que se vende o compra.
asiento El asiento contable. Se relaciona con un ejercicio y se compone de partidas.
asiento_factura Esta clase permite genera un asiento a partir de una factura.
balance Define que cuentas hay que usar para generar los distintos informes contables.
balance_cuenta Detalle de un balance.
balance_cuenta_a Detalle abreviado de un balance.
banco Entidad bancaria.
caja Estructura para almacenar los datos de estado de una caja registradora (TPV).
cliente El cliente. Puede tener una o varias direcciones y subcuentas asociadas.
concepto_partida Un concepto predefinido para una partida (la línea de un asiento contable).
cuenta Elemento de tercer nivel del plan contable. Está relacionada con un único ejercicio y epígrafe, pero puede estar relacionada con muchas subcuentas.
cuenta_banco Una cuenta bancaria de un cliente.
cuenta_banco_cliente Una cuenta bancaria de un cliente.
cuenta_banco_proveedor Una cuenta bancaria de un proveedor.
cuenta_especial Permite relacionar cuentas especiales (VENTAS, por ejemplo) con la cuenta o subcuenta real.
direccion_cliente Una dirección de un cliente. Puede tener varias.
direccion_proveedor Una dirección de un proveedor. Puede tener varias.
divisa Una divisa (moneda) con su símbolo y su tasa de conversión respecto al euro.
ejercicio Ejercicio contable.
empresa La empresa.
epigrafe Segundo novel del plan contable.
factura_cliente Factura de un cliente.
factura_proveedor Factura de un proveedor.
familia Una familia de artículos (el equivalente a la marca del artículo).
forma_pago Forma de pago de una factura.
fs_access Define que un usuario tiene acceso a una página concreta.
fs_button Botón de la cabecera.
fs_button_img Botón de la cabecera con imagen.
fs_cache Clase para concectar e interactuar con memcache.
fs_controller La clase principal de la que deben heredar todos los controladores (las páginas) de FacturaScripts.
fs_db Esta es la clase de la que heredan las clases fs_mysql y fs_postgresql.
fs_default_items Esta clase sólo sirve para que los modelos sepan que elementos son los predeterminados para la sesión. Pero para guardar los valores hay que usar las funciones fs_controller::save_lo_que_sea()
fs_extension Permite que un plugin añada elementos a una vista existente, sin necesidad de un reemplazo completo.
fs_log Clase para almacenar el historial de acciones de los usuarios.
fs_model La clase de la que heredan todos los modelos, conecta a la base de datos, comprueba la estructura de la tabla y de ser necesario la crea o adapta.
fs_mysql Clase para conectar a MySQL
fs_page Elemento del menú de FacturaScripts.
fs_pdf Permite la generación de PDFs algo más sencilla.
fs_postgresql Clase para conectar a PostgreSQL
fs_printer Permite imprimir ticket de forma local o remota.
fs_user Usuario de facturaScripts. Puede estar asociado a un agente.
fs_var Una clase genérica para consultar o almacenar en la base de datos pares clave/valor.
grupo_clientes Un grupo de clientes, que puede estar asociado a una tarifa.
grupo_epigrafes Primer nivel del plan contable.
impuesto Un impuesto (IVA) que puede estar asociado a artículos y línes de albaranes o facturas.
linea_albaran_cliente Línea de un albarán de cliente (boceto de factura).
linea_albaran_proveedor Línea de un albarán de proveedor (boceto de una factura).
linea_factura_cliente Línea de una factura de cliente.
linea_factura_proveedor Línea de una factura de proveedor.
linea_iva_factura_cliente La línea de IVA de una factura de cliente. Indica el neto, iva y total para un determinado IVA y una factura.
linea_iva_factura_proveedor La línea de IVA de una factura de proveedor. Indica el neto, iva y total para un determinado IVA y una factura.
pais Un país, por ejemplo España.
partida La línea de un asiento. Se relaciona con un asiento y una subcuenta.
proveedor Un proveedor. Puede estar relacionado con varias direcciones o subcuentas.
regularizacion_iva Una regularización de IVA.
secuencia Estos tres modelos (secuencia, secuencia_contabilidad y secuencia_ejercicio) existen para mantener compatibilidad con eneboo, porque maldita la gana que yo tengo de usar TRES tablas para algo tan simple...
secuencia_contabilidad Clase que permite la compatibilidad con Eneboo.
secuencia_ejercicio Clase que permite la compatibilidad con Eneboo.
serie Una serie de facturación o contabilidad, para tener distinta numeración en cada serie.
stock La cantidad en inventario de un artículo en un almacén concreto.
subcuenta El cuarto nivel de un plan contable. Está relacionada con una única cuenta.
subcuenta_cliente Relaciona a un cliente con una subcuenta para cada ejercicio
subcuenta_proveedor Relaciona a un proveedor con una subcuenta para cada ejercicio
sucursal La sucursal de un banco. Está relacionada con un único banco.
tarifa Una tarifa para los artículos.
tarifa_articulo El precio concreto de un artículo en una tarifa determinada.
